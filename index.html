About
The multivariate grade calculator aims to make it easy to evaluate numerous possible combinations in order to achieve a desired grade. The grade thermometer on the right shown in the demo displays the minimum and maximum possible final grade the user will achieve based on what they entered as well as their predicted final score.

The grading policy used in the demo is as follows:
  • 50% — Three exams (all equally weighted)
  • 40% — 10 programming assignments (all equally weighted)
  • 10% — Mini-project

You can try using the multivariate grade calculator here.

The webpage was written with pure HTML, CSS and JavaScript without the use of any libraries or frameworks which regrettably resulted in many sloc. You may find the source code here.

An initial version was written in python using File I/O, NumPy and Matplotlib. You may find the source code here.

Walking Through the Numbers

Using the same grading policy from the about section, assume that the following scores have been entered already.
Exam: 82%
Programming assignments (PAs): 80%, 84%, 65%, 70%

Under PAs information
Total assessments completed: 4 / 10

  • 4 out of 10 total assignments have been done
Current standing: 299.00 / 400 = 74.75%
  • 80 + 84 + 65 + 70 = 299
  • Average score so far is 74.75%
Final standing: 299.00 / 400 = 29.90%
  • Final average score if you do not turn in any more PAs
Overall percentage weighting: 40.00%
  • Percent weighting of PAs towards final grade
You had the opportunity to earn: 16.00%
  • With each PA being worth 4%, 4 were completed so far --> 4% * 4 = 16%
Secured percentage: 11.96%
  • With each PA being worth 4% --> 0.8*4 + 0.84*4 + 0.65*4 + 0.7*4 = 11.96%
  • The percent you gained towards your final grade
Permanent loss: 4.04%
  • 16% (possibly earnt) - 11.96% (earnt) = 4.04%
  • The percent you lost so far
